Install & setup guide

How to download, set up and run DeryaOS — written for people who have never used a virtual machine before. Pick the path that fits you.

Easiest: VMware / VirtualBoxQEMUReal PC install

Before you start. DeryaOS is early beta software. Running it inside a virtual machine (below) is completely safe for your computer. Only the “real PC install” section touches a physical disk — don’t do that on a machine whose data you can’t afford to lose.

What you’ll need

  • A Windows, macOS or Linux computer with about 2 GB of free RAM and 1 GB free disk.
  • One of: VMware Workstation/Player, VirtualBox, or QEMU (all free). The guide shows each.
  • A few minutes. No account or sign-up is needed to download.

All downloads are on the DeryaOS releases page.

A · Run it in VMware or VirtualBox — easiest, recommended

This uses a ready-made “appliance” file (.ova). You import it once and DeryaOS boots — nothing to install or configure.

  1. Get a virtual-machine app (if you don’t have one):
    • VirtualBox — free, all platforms: virtualbox.org/wiki/Downloads
    • VMware Workstation Player — free for personal use: vmware.com
  2. Download the appliance: deryaos-x86.ova (about 7 MB).
  3. Import it:
    • VirtualBox → menu File → Import Appliance… → choose the .ova → Next → Finish.
    • VMware → menu File → Open… → choose the .ova → give it a name → Import. (If VMware warns the appliance “did not pass OVF specification conformance”, click Retry — it imports fine.)
  4. Start it: select the new DeryaOS machine and click Start (the green ▶). DeryaOS boots in a window.
Tip: click inside the window to give it your mouse/keyboard; press the host key (VirtualBox: Right Ctrl; VMware: Ctrl+Alt) to release it.

B · Run it in QEMU

QEMU is a free, lightweight emulator. We ship a zip with the disk image and one-click launchers.

  1. Install QEMU: qemu.org/download (pick your operating system; on Windows use the installer, on macOS brew install qemu, on Linux your package manager).
  2. Download the VM bundle: deryaos-x86-vm.zip and unzip it anywhere.
  3. Run the launcher inside the unzipped folder:
    • Windows — double-click run-vm-windows.bat
    • Linux / macOS — open a terminal in the folder and run ./run-vm-linux.sh

Prefer to type it yourself? The launcher just runs:

qemu-system-x86_64 -drive file=deryaos.img,format=raw -m 512 \
  -netdev user,id=n0 -device e1000,netdev=n0 -rtc base=localtime -display gtk

C · Install on a real PC

This writes to a real disk. Use a spare machine or one whose data you’ve backed up. You have been warned.
  1. Download the installer image: deryaos-x86.img.gz, then un-gzip it to get deryaos.img (Windows: 7-Zip; Linux/macOS: gunzip deryaos-x86.img.gz).
  2. Write it to a USB stick (this erases the stick):
    • Windows — use Rufus: select the USB, choose deryaos.img, write in DD image mode.
    • Linux/macOS — sudo dd if=deryaos.img of=/dev/sdX bs=4M status=progress (replace /dev/sdX with your USB device — double-check it!).
  3. Boot the target PC from the USB (open its boot menu — usually F12/F11/Esc at power-on). DeryaOS comes up at a text shell.
  4. Install to the internal disk from that shell: 
    inst plan ATA0            # survey the disk (try AHCI0 / NVME0 if ATA0 is empty)
inst -f ATA0 compact     # fresh full install to that disk
    Then reboot and pick the new disk. Tiny CD/diskette alternative: the seed ISO and 1.44 MB diskette boot the same installer.

First steps after it boots

  • If it starts in text mode, you’ll see a deryaos:C:/ $ prompt. Useful commands: help, ver (version), ls (list files), gui (start the desktop where available).
  • On the desktop: a taskbar with a Start menu, draggable windows, and built-in apps — a text editor, file manager, web viewer, calculator, clock, system tools and a few games.
  • Networking is on by default in the VM; the web viewer and update tools can reach the internet through it.

Updates (over-the-air)

DeryaOS can update itself: it fetches new components in the background and verifies each one cryptographically (Ed25519 + SHA-256) before applying it, rolling back automatically if a build misbehaves. To check for updates from the shell:

osupdate check
osupdate sync

For now, the simplest way to get the newest build is to grab the latest VM or image from the releases page. A hosted update service is being brought online; once it’s live, the VMs will pick up new builds on their own.
